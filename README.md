## 🌟 V2.6.1 核心更新 (2026.03)

1. **异步数据库架构优化**：
   - 全面升级为 AsyncSession 异步数据库操作，彻底解决同步调用导致的性能瓶颈
   - PortfolioManager 和 OrderExecutor 完全异步化，支持高并发场景
   - 修复所有 `RuntimeWarning: coroutine was never awaited` 警告
2. **代码库清理**：
   - 删除所有临时测试文件 (test_*.py)
   - 删除所有诊断工具文件 (diagnose_*.py)
   - 保持代码库整洁，便于维护

## 🌟 V2.6 核心更新 (2026.03)

1. **量价动能引擎 (Momentum & Volume Engine)**：
   - 全面引入 **ROC (变动率)** 和 **OBV (能量潮)** 双动能因子，彻底解决系统偏好抄底、容易踏空主升浪的资金内耗问题。
   - 趋势追入强制要求量价齐升 (OBV 确认)，有效过滤缩量诱多陷阱。
2. **动态资金打分系统重构**：
   - 摒弃了单一的 `100 - RSI` 抄底打分机制，针对右侧突破信号采用 `基础分 + ADX趋势强度 + ROC加速度` 的多因子竞价模型。
   - 强势资产将优先获得资金槽位，真正实现“强者恒强”的投资组合分配。
3. **杠杆 ETF 专属风控升级**：
   - 动量突破入场由固定打分升级为动态打分，增强主升浪捕捉能力。
   - 完善基于 ADX 的自适应跟踪止盈，在震荡市收紧防守，在单边市放宽波动以博取大利。

## 🌟 V2.5 核心更新 (2026.03)

1. **预算槽位管理 (Budget Slotting)**：
   - 彻底解决“小资金多资产”下的加仓冲突问题。系统将总资产平分为若干“槽位” (max_concurrent_assets)，为每个已持仓资产**强制预留**后续网格加仓资金。
   - 严格遵守全局 10% 现金储备底线，确保极端行情下的补仓能力。
2. **金融级并发安全 (Concurrency Protection)**：
   - 引入 **GlobalTradeLock (全局交易锁)**：防止定时任务与实时止损线程在执行下单、更新余额时产生竞态冲突。
   - **原子化数据库更新**：采用 SQL 级 `UPDATE balance = balance + delta` 操作，确保多线程环境下的账面金额 100% 准确。
3. **实时持仓透视 (Live Holdings)**：
   - `holdings` 表新增 `last_price` 实时同步。高频报价线程每 5 秒自动更新所有持仓的最新市价与浮动盈亏 (P&L)。
   - **Board Lot 智能持久化**：自动缓存标的的“每手股数”到数据库，极大降低 Futu API 频率消耗。
4. **2026 生产基准更新**：
   - 全面更新 **2026 年 A股/港股法定节假日** 逻辑，确保分析任务在非交易日精准挂起。
   - 优化后台绘图引擎，支持在无 GUI 环境下的多线程异步生成分析图表。

### 📈 个股策略 (趋势跟踪)

#### 核心理念
- **双重风控**: ATR 动态止损 + 5% 极值回撤止盈
- **智能再入场**: 解决牛市“踏空”问题的 Strong Trend Re-entry 机制
- **风险控制**: 严格的止损止盈机制

#### 买入条件 (需满足 ≥2 个条件)

```python
# 条件1: 均线金叉 + 放量突破
if (SMA_5 > SMA_20) and (Volume > VOL_SMA_5):
    score += 1

# 条件2: RSI超卖 + 趋势向上
if (RSI_14 < 35) and (Price > SMA_200):
    score += 1

# 条件3: 布林带下轨支撑 + 非下跌趋势
if (Price <= BOLL_LOWER * 1.01) and (Price > SMA_200):
    score += 1
```

#### 卖出条件 (需满足 ≥2 个条件)

```python
# 条件1: 均线死叉
if SMA_5 < SMA_20:
    score += 1

# 条件2: RSI超买 + 非强趋势
if (RSI_14 > 70) and (ADX <= 25):
    score += 1

# 条件3: 布林带上轨压力 + 非强势上涨
if (Price >= BOLL_UPPER * 0.99) and (ADX <= 25):
    score += 1
```

### 📊 ETF策略 (网格交易)

#### 核心理念
- **槽位化预算管理**: V2.5 引入。将总资金划分为槽位，为每个 ETF 预留 4 批次加仓资金。
- **全局总仓位风控**: 全系统总持仓强制闭锁在 90% 以内，保留 10% 现金缓冲应对市场剧震。
- **动态追踪止盈**: 5% 跌幅自动锁利，盈利峰值实时动态更新。
- **均值回归**: 利用布林带和 RSI 极限值进行高频波动获利

#### 建仓策略 (4批次倒金字塔)

```

加仓1   20%        距首仓成本下跌3%             < 35
加仓2   20%        距加仓1成本下跌3%            < 35
```

## ❓ 常见问题

### 🔌 连接问题

#### Q: 富途API连接失败 (ECONNREFUSED)
**解决方案**: 检查 FutuOpenD 是否在 11111 端口运行，且是否已登录并开启了“允许 API 访问”。

### 💰 资金管理问题

#### Q: 为什么有现金但不买入新资产？
**原因**: V2.5 引入了槽位管理。如果 `max_concurrent_assets` 已满，系统会为现有资产预留加仓资金，拒绝开新仓。请调整配置中的并发上限。
